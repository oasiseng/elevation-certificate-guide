# Contributing to the Elevation Certificate Guide

Thank you for your interest in improving the most comprehensive open-source resource on FEMA Elevation Certificates. Contributions from practitioners, floodplain managers, insurance professionals, and property owners make this resource better for everyone.

## How to Contribute

### Reporting Issues

If you find inaccurate information, outdated references, or gaps in coverage, please open an issue with:

- A clear description of the problem
- The specific file and section affected
- The correct information (with a source if possible)
- Your professional context (e.g., "I'm a floodplain manager in Florida and this doesn't match our local requirements")

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b improve-crawlspace-section`)
3. Make your changes
4. Ensure your content is accurate and sourced where possible
5. Submit a pull request with a clear description of what you changed and why

### What We're Looking For

- **State-specific guidance**: NFIP requirements interact with state laws differently. If you can add state-specific notes, that's extremely valuable.
- **Real-world examples**: Anonymized case studies of tricky EC situations (split-levels, sunken rooms, unusual foundations) help practitioners learn.
- **Updated FEMA references**: When FEMA updates forms, bulletins, or procedures, help us keep the docs current.
- **Community-specific practices**: How different jurisdictions handle EC requirements, CRS programs, and local ordinances.
- **Corrections**: If you spot an error in flood zone descriptions, form field guidance, or regulatory references, please flag it.

### Style Guidelines

- Write in clear, professional prose
- Avoid jargon without explanation
- When referencing regulations, cite the specific CFR section (e.g., 44 CFR 60.3(c)(5))
- When referencing the EC form, use the current form number (FEMA Form FF-206-FY-22-152)
- Keep audience-specific guides accessible to their target audience
- Use markdown formatting consistently with the existing docs

### What Doesn't Belong

- Marketing content for specific companies or products
- Legal advice or insurance rate guarantees
- Proprietary procedures that can't be shared publicly
- Content that contradicts current FEMA guidance without clear explanation

## Code of Conduct

Be professional, respectful, and constructive. This is a resource built by practitioners for practitioners. Disagreements about technical interpretations are welcome — keep them focused on the facts and cite your sources.

## Questions?

Open a discussion thread if you're unsure whether your contribution fits. We'd rather help you shape it than miss a valuable addition.
