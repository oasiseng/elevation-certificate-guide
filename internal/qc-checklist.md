# Quality Control Checklist for Elevation Certificates

**Template review checklist — customize for your firm's QC process**

---

## Overview

This checklist is designed to be completed by a QC reviewer (typically a licensed professional different from the certifier) before an Elevation Certificate is released to the client or community. The checklist covers all sections of FEMA Form 086-0-33 and common review points from hundreds of completed ECs.

Use this as a paper checklist or import into your firm's document review system. Customize the standards and thresholds to match your firm's specific practices and regional conditions.

---

## Pre-Review Setup

- [ ] **EC Form retrieved** and all pages present (Sections A, B, C, D, and photographs)
- [ ] **Field notes available** for comparison
- [ ] **Property address identified** — Review EC, field notes, and FIRM for consistency
- [ ] **Prior ECs checked** — Does this update a previous certificate? Note prior findings
- [ ] **Reviewer credentials confirmed** — Reviewer is licensed professional (engineer or surveyor) qualified to approve ECs for this state
- [ ] **Conflict checked** — Reviewer is independent of certifier (not same person)

## Section A: Building and Property Information

### Owner and Property Identity

- [ ] **Owner name** matches property records (tax assessor, deed, or client provided info)
- [ ] **Owner name matches field notes** (compare handwriting, abbreviations)
- [ ] **Mailing address complete** — Street, city, state, ZIP code
- [ ] **Unit/Apartment/Building number included** (if applicable to multi-unit complex)
- [ ] **Property description** (legal description, tax parcel ID, or other identifier) is accurate per county records
- [ ] **Property description matches field notes** — No discrepancies in lot number or section

### Building Identification

- [ ] **Building address** matches FIRM search location and field notes
- [ ] **Address appears on all pages** of the EC (Section C header or footer confirms address)
- [ ] **Building use** correctly identified:
  - [ ] Residential (single-family, multi-family, condominium, townhouse)
  - [ ] Commercial, industrial, institutional
  - [ ] Mixed-use
- [ ] **Building use description** in field notes matches form selection
- [ ] **Coordinates (latitude/longitude)** are reasonable for the stated address
  - *Tip: Verify using Google Maps or GIS tool; latitude/longitude should be within 50 feet of property center*
- [ ] **Coordinate datum** is checked or assumed (typically WGS 84 for GPS)

### Building Diagram

- [ ] **Building diagram number** selected (Diagram 1–7 or applicable custom diagram)
- [ ] **Diagram matches field photographs** — Review photos for confirmation:
  - [ ] Foundation type visible (slab, crawlspace, basement, elevated)
  - [ ] Number of visible stories matches diagram
  - [ ] Garage presence/absence matches diagram selection
  - [ ] Any special features visible (deck, breakaway wall, split-level)?
- [ ] **Diagram is current FEMA version** — Not an outdated diagram from old EC guidance
- [ ] **If diagram deviation noted** — Is explanation documented in Section D? (e.g., "Diagram 7 standard crawlspace, but house has interior sunken room")

### Crawlspace/Enclosure Information (A8a–f)

If building has crawlspace or enclosed basement, verify:

- [ ] **Crawlspace existence confirmed** — Field photos show crawlspace access or vents
- [ ] **Square footage (A8a) reasonable** — Compare to field measurements:
  - [ ] Width and length recorded in field notes
  - [ ] Calculation verified: Length × Width = Sq ft
  - [ ] For irregular crawlspace: breakout into rectangles and summed correctly
  - [ ] Result is plausible for building footprint (roughly 70–90% of main building sq ft)
- [ ] **A8b — Number of openings on lowest side**:
  - [ ] Count matches field observations (field notes and photos)
  - [ ] Openings are flood vents, louvers, or grates — not HVAC returns or electrical conduits
  - [ ] At least 2 openings per side if non-engineered (FEMA minimum requirement)
- [ ] **A8c — Number of openings on perpendicular side**:
  - [ ] Count matches field observations on second side
  - [ ] Consistent with non-engineered requirement (≥2 openings)
- [ ] **A8d — Height of bottom of opening above adjacent grade**:
  - [ ] Measurement is to **bottom of vent**, not top
  - [ ] Height is reasonable (typically 6–24 inches for residential crawlspace vents)
  - [ ] Height is above adjacent grade (not below — would trap water)
  - [ ] Measurement source documented (field notes show specific vent measured)
- [ ] **A8e — Net open area (up to 3 openings)**:
  - [ ] Gross opening dimensions (width × height) recorded in field notes
  - [ ] Net open % estimated (typical vent: 40–60% open; screen reduces by 10–20%)
  - [ ] Calculation verified: Gross area × Net % = Net open area
  - [ ] Net open area is in square inches (not feet — common error)
  - [ ] Total net open area meets minimum requirement:
    - *FEMA guidance: 1 sq in per 1 sq ft of crawlspace; local codes may differ*
    - *Example: 500 sq ft crawlspace requires ≥500 sq in total net open area*
  - [ ] If only 1–2 openings listed, verify remaining openings were too small to list (document in notes)
- [ ] **A8f — Engineered or Non-Engineered**:
  - [ ] Selection matches field observations
  - [ ] **Non-engineered**: Standard builder vents, no design documentation; ≥2 openings per side
  - [ ] **Engineered**: Design specifications in building plans; engineer-stamped drawings available
  - [ ] If engineered, design documentation referenced or attached

### Attached Garage Information (A9a–f)

If building has attached garage:

- [ ] **Garage existence confirmed** — Field photos show garage door entry
- [ ] **A9a — Square footage reasonable**:
  - [ ] Width and length recorded in field notes
  - [ ] Calculation verified: Length × Width = Sq ft
  - [ ] Result plausible (typical 2-car: 400–500 sq ft; 3-car: 600–700 sq ft)
- [ ] **A9b–f — Flood openings** (if applicable):
  - [ ] If garage is above BFE or not in flood zone: openings may be N/A — document assumption
  - [ ] If garage is in floodplain with standard garage door (does not seal): opening calculations performed
  - [ ] Opening calculations follow same rules as crawlspace (A8d–f)
  - [ ] Rollup door counted as opening (estimate bottom height and opening area)
- [ ] **A9 marked "N/A"** if garage is detached, interior (not accessible from exterior), or does not exist

### Photographs (A10)

- [ ] **Minimum 2 photographs present** (ideally 4: front, left, rear, right)
- [ ] **Foundation clearly visible** in at least 2 photos:
  - [ ] Slab edge or crawlspace stem wall shows grade transition
  - [ ] Relationship between foundation and above-ground structure visible
  - [ ] Foundation material (concrete, masonry, wood) identifiable
- [ ] **Flood vents/openings visible** in close-up or wide-angle view:
  - [ ] Vent louver, grate, or opening shown
  - [ ] Relationship of vent bottom to adjacent ground visible
  - [ ] Vent type identifiable (round, rectangular, foundation wall opening)
- [ ] **Building address visible** on photo or labeled
- [ ] **Photo date recorded** — Label on back or caption indicates photo date
- [ ] **View direction labeled** — Front, Left Rear, Right (or N/S/E/W)
- [ ] **Photo quality acceptable** — Not blurry, too dark, or washed out
- [ ] **Photos match building type** — Photos confirm building diagram and observations

## Section B: FIRM and Flood Data

### Community and Map Identification

- [ ] **NFIP Community Name** correct and spelled accurately
- [ ] **NFIP Community Number** is 4-digit code matching official FEMA list
- [ ] **County name** correct per state and FEMA Community list
- [ ] **State abbreviation** correct
- [ ] **Map Panel Number** matches current effective FIRM:
  - [ ] Panel number format verified (e.g., "0340" or "0001")
  - [ ] Suffix (if present) indicates revision (e.g., "A", "B", "Rev. 04/15/15")
- [ ] **FIRM Index Date** matches community's reference map
  - *This is the date of the community FIRM index, not the effective date of the panel*
- [ ] **FIRM Panel Effective/Revised Date** is current:
  - [ ] Is this the most recent effective panel for the community?
  - [ ] Reviewer checked FEMA Map Service Center or state emergency management office
  - [ ] If panel is >5 years old, note reason (e.g., community has not updated FIRM recently)

### Flood Zone and Elevation Data

- [ ] **Flood Zone** matches FIRM at subject property:
  - [ ] Zone AE, A (with BFE), AO, AR, A (no BFE), A99, X (shaded), X (unshaded), V, VE, etc.
  - [ ] If uncertain, cross-check with FEMA Map Service Center
- [ ] **Base Flood Elevation (BFE)** is documented:
  - [ ] BFE value appears in Section B and Section C (elevation datum section)
  - [ ] BFE is reasonable for property elevation (should be close to, but slightly lower than, measured first-floor elevation)
  - [ ] BFE source identified: FIS, FIRM, Community Determined, Other
- [ ] **BFE Datum** specified and matches elevations in Section C:
  - [ ] Datum is NGVD 29, NAVD 88, or other recognized datum
  - [ ] If Section C uses different datum than BFE, conversion factor documented in Section D
  - [ ] Conversion factor source verified (NOAA, USGS, state DOT)
- [ ] **Base Flood Depth (for AO zones)** documented if applicable:
  - [ ] Instead of BFE, Zone AO areas list average depth of flooding (e.g., "1–3 feet")
  - [ ] Value is reasonable for the area

### Additional FIRM Data

- [ ] **LiMWA (Levee Impacted Modified Work Area) status checked**:
  - [ ] If "Yes": Property is near a levee; discuss implications with client
  - [ ] If "No": Not applicable
- [ ] **CBRS/OPA (Coastal Barrier Resources System / Otherwise Protected Area) status checked**:
  - [ ] If "Yes": Property may be in protected coastal area; certain NFIP limitations apply
  - [ ] If "No": Not applicable
- [ ] **Coastal High Hazard Zone (V zone)** noted if applicable:
  - [ ] If property is in VE or V zone: Is bottom-of-lowest-horizontal-member measurement particularly critical?
  - [ ] Are additional certifications or documentation required by state or community?

## Section C: Elevation Data

### Construction Status and Benchmark

- [ ] **Construction status** selected (Drawings, Under Construction, Finished):
  - [ ] "Finished" typically applies to existing structures (most ECs)
  - [ ] "Drawings" or "Under Construction" used only for new buildings before completion
- [ ] **Benchmark identification** is specific and verifiable:
  - [ ] Benchmark named (e.g., "USGS BM 12345-67-8910" or "Mailbox post SE corner, reference mark on top")
  - [ ] If published NGS benchmark: Reference number matches USGS Datasheet
  - [ ] If temporary benchmark: Description includes location, marking method, and reference photo
  - [ ] Benchmark elevation documented in both absolute terms and relative to subject property
- [ ] **Benchmark location reasonable** — Within 500–1000 feet of property or tied through known control points

### Elevation Datum and Conversion

- [ ] **Vertical Datum for Elevations** (Section C, upper right):
  - [ ] Datum specified: NGVD 29, NAVD 88, State Plane, or Other
  - [ ] Datum matches BFE datum in Section B
  - [ ] **If datum differs from BFE**: Conversion factor documented in Section D
- [ ] **Datum Conversion (if applicable)**:
  - [ ] If Section C uses different datum than BFE, conversion factor clearly noted
  - [ ] Conversion source identified: NOAA, USGS, state DOT, or NGS publication
  - [ ] Conversion factor value is reasonable (typically ±0.5 feet for NGVD 29 to NAVD 88)
  - [ ] Direction of conversion noted (e.g., "NAVD 88 elevation = NGVD 29 + 0.3 feet")
  - [ ] All elevations in Section C interpreted with datum in mind

### Individual Elevation Measurements (C2.a through C2.h)

#### C2.a — Top of Bottom Floor

- [ ] **Measurement location matches building type**:
  - [ ] Slab-on-grade: Top surface of concrete slab
  - [ ] Crawlspace: Top of rim joist or foundation wall, NOT joist on bottom of floor structure
  - [ ] Basement: Top of floor slab or rim (per applicable building code)
  - [ ] Elevated: Top of floor joists or platform
- [ ] **Multiple measurements recorded in field notes** (at least 3 locations):
  - [ ] Corners and midpoint of long wall measured
  - [ ] Lowest value correctly identified and reported
- [ ] **Elevation is reasonable**:
  - [ ] For slab: C2.a ≈ LAG (within 0.1–0.3 feet; slight slope expected)
  - [ ] For crawlspace: C2.a is notably higher than LAG (typically 2–4 feet above LAG)
  - [ ] If BFE known: First floor is usually above BFE (per code)
  - [ ] No obvious transposition (e.g., elevation of 24.3 when it should be 234.3)

#### C2.b — Top of Next Higher Floor (if applicable)

- [ ] **Applicable only if building is multi-story**:
  - [ ] Single-story building: C2.b is blank or marked "N/A"
  - [ ] Two-story: C2.b measured and documented
- [ ] **Measurement is to top of second floor joists or rim**:
  - [ ] Not to finished ceiling or insulation
  - [ ] At least 2 locations measured; lowest value reported
- [ ] **Elevation is reasonable**:
  - [ ] C2.b is typically 8–10 feet higher than C2.a (floor-to-floor height)
  - [ ] C2.b is higher than C2.a (not equal or reversed)

#### C2.c — Bottom of Lowest Horizontal Structural Member

- [ ] **Applicable to all buildings, especially V zones**:
  - [ ] Identified in field notes (rim joist, truss chord, roof rafter, etc.)
  - [ ] Lowest point of structure across entire building footprint located
- [ ] **V Zone and Coastal A Zone Properties: Critical Measurement**:
  - [ ] If property is in V zone or coastal A zone (with V zone notation):
    - [ ] Bottom-of-lowest-horizontal-member is often the certifying elevation
    - [ ] Is this elevation significantly higher than top of bottom floor (C2.a)?
    - [ ] If difference is <0.5 feet, review field notes for accuracy
- [ ] **Elevation is reasonable**:
  - [ ] C2.c is either equal to or higher than C2.a (structures don't invert)
  - [ ] For elevated structures, C2.c may be significantly higher than C2.a
  - [ ] For slab buildings, C2.c is typically rim joist height (1–2 feet above C2.a)

#### C2.d — Top of Garage Floor Slab (if applicable)

- [ ] **Correctly marked as applicable or N/A**:
  - [ ] Attached garage: C2.d filled in
  - [ ] Detached garage or no garage: C2.d blank or "N/A"
  - [ ] If garage is integral with main building (shared slab): C2.d ≈ C2.a (within 0.2 feet)
- [ ] **Measurement is to top of slab**:
  - [ ] Not to finished flooring (carpet, tile, etc.)
  - [ ] At least 2 locations measured; lowest value reported
- [ ] **Elevation is reasonable**:
  - [ ] C2.d is approximately equal to C2.a if same slab
  - [ ] If garage is separate slab: C2.d should be similar to main building first floor (not significantly different unless intentional slope)

#### C2.e — Lowest Elevation of M&E (Mechanical and Electrical)

- [ ] **All significant systems measured or documented**:
  - [ ] HVAC units (interior and exterior)
  - [ ] Water heater
  - [ ] Furnace
  - [ ] Electrical panel
  - [ ] Gas meter
  - [ ] Propane tank
  - [ ] Other equipment relevant to flood damage
- [ ] **Inside vs. outside documented**:
  - [ ] Clarifies whether equipment is interior (e.g., attic HVAC) or exterior (pad-mounted A/C unit)
  - [ ] Outdoor pad-mounted units typically lower than indoor equipment
- [ ] **Lowest value correctly identified**:
  - [ ] If multiple units: lowest one reported in C2.e
  - [ ] Field notes confirm which unit is lowest and why
- [ ] **Elevation is reasonable**:
  - [ ] C2.e is typically at or slightly below C2.a (equipment often near slab level or in crawlspace)
  - [ ] Outdoor equipment pads measured to their top surface
  - [ ] If C2.e is significantly above C2.a, document reason (e.g., "furnace in attic, lowest equipment is water heater at 235.5 feet")
- [ ] **Section D includes description**:
  - [ ] Equipment types and locations listed in comments (required)
  - [ ] Clarifies any equipment above C2.a elevation

#### C2.f — Lowest Adjacent Grade (LAG)

- [ ] **Properly defined as Natural or Finished Grade**:
  - [ ] Natural: Pre-construction ground elevation (rarely measured; use pre-development topo if available)
  - [ ] Finished: Current ground elevation after landscaping and improvements (typical selection)
- [ ] **Multiple measurements taken**:
  - [ ] Grade measured on all four sides of building
  - [ ] At least 3 locations per side, within 1 foot of foundation
- [ ] **Lowest value correctly reported**:
  - [ ] LAG is minimum elevation from all measurements
  - [ ] Not an average (very common error)
- [ ] **Elevation is reasonable**:
  - [ ] For slab-on-grade: LAG ≈ C2.a (within 0.1–0.3 feet; slope acceptable)
  - [ ] For crawlspace: LAG is noticeably lower than C2.a (typically 2–4 feet difference)
  - [ ] For elevated on piers: LAG is significantly lower than C2.a (6–12+ feet)
  - [ ] LAG is lower than HAG (property slopes up on at least one side, or is flat)

#### C2.g — Highest Adjacent Grade (HAG)

- [ ] **Correctly identified**:
  - [ ] Highest point of ground immediately adjacent to building
  - [ ] Not a distant hilltop or feature far from structure
  - [ ] Usually within 5–50 feet of building
- [ ] **Measurement documented**:
  - [ ] Field notes identify where HAG was measured (e.g., "SE corner, upslope side")
  - [ ] HAG location makes sense relative to property and neighborhood topography
- [ ] **Elevation is reasonable**:
  - [ ] HAG ≥ LAG (always equal or higher than LAG)
  - [ ] HAG – LAG = slope of property
    - [ ] Flat property: HAG ≈ LAG (difference <0.5 feet)
    - [ ] Sloped property: HAG is notably higher (1–10+ feet difference typical)
  - [ ] HAG is documented as Natural or Finished grade (matching LAG designation)

#### C2.h — Lowest Elevation of Deck/Stairs with Structural Support

- [ ] **Applicability determined correctly**:
  - [ ] Building has attached deck with ledger bolts: C2.h filled in
  - [ ] Deck is detached or independent footings: C2.h marked "N/A"
  - [ ] Stairs are free-standing (not attached): C2.h marked "N/A"
  - [ ] No deck or stairs: C2.h blank or "N/A"
- [ ] **Measurement is to lowest structural member**:
  - [ ] Top of joist, beam, or slab supporting deck/stairs
  - [ ] Not to finished decking or railing
- [ ] **Elevation is reasonable**:
  - [ ] C2.h is typically equal to or slightly lower than C2.a (deck attached to building)
  - [ ] If C2.h is significantly higher than C2.a, document reason in Section D

### Measurement Units

- [ ] **All elevations in same units**:
  - [ ] Feet (ft) used throughout in US (typical)
  - [ ] Meters (m) if international or specified by client
  - [ ] Not a mix of feet and meters — common source of downstream errors
- [ ] **Unit clearly stated** on form or in Section D

### Reasonableness Check (All Elevations)

- [ ] **Elevations are within expected range for building type**:
  - [ ] Typical single-story slab: 230–250 feet (in coastal areas; adjust for region)
  - [ ] Typical single-story crawlspace: LAG 225–235 feet, C2.a 232–242 feet
  - [ ] Multi-story building: appropriate floor-to-floor spacing (8–10 feet)
- [ ] **No obvious data entry errors**:
  - [ ] Decimal points in correct location
  - [ ] No transposed digits (e.g., 234.3 vs. 324.3)
  - [ ] Field notes support reported values
- [ ] **Consistency across measurements**:
  - [ ] Related elevations make sense (C2.a < C2.b < C2.c in multi-story)
  - [ ] M&E elevation (C2.e) is lower than or equal to C2.a (equipment doesn't float above building)
  - [ ] LAG < HAG (property slopes up or is flat, never reverses)

## Section D: Certifier Information and Comments

### Certifier Credentials

- [ ] **Certifier name** legible and spelled correctly
- [ ] **Professional license number** is present and format appropriate:
  - [ ] PE (Professional Engineer): 6–8 digit number typical
  - [ ] PLS (Professional Land Surveyor): similar format per state
  - [ ] License number is state-specific (e.g., "CA PE #12345" or "TX LS #1234")
- [ ] **Discipline/Title** indicates qualified professional:
  - [ ] Professional Engineer, Professional Surveyor, Registered Architect (if authorized by state)
  - [ ] Not a technician, intern, or unlicensed person
- [ ] **Company name** and contact information complete:
  - [ ] Company name
  - [ ] Office address
  - [ ] Phone and/or email
- [ ] **Company appears to be legitimate**:
  - [ ] Not a generic or suspicious entity
  - [ ] Consistent with known surveying/engineering firms
  - [ ] Can be verified if needed

### Signature, Seal, and Certification

- [ ] **Certifier signature** is present:
  - [ ] Ink signature (or electronic signature if permitted by state)
  - [ ] Signature appears to match certifier's style if known
- [ ] **Professional seal/stamp** affixed:
  - [ ] Seal is clear and legible
  - [ ] Seal includes PE or PLS designation
  - [ ] Seal includes state and license number
  - [ ] Seal is original (not photocopy of seal, if possible to verify)
- [ ] **Date signed** is present:
  - [ ] Date is reasonable (recent, not months after measurement)
  - [ ] Date is after field work was completed (field notes earlier than signature date)
- [ ] **Certification statement** affirms accuracy:
  - [ ] Standard FEMA language or equivalent in Section D or signature block
  - [ ] Certifier acknowledges responsibility for accuracy

### Latitude/Longitude Surveyor Question

- [ ] **Question answered**: "Did you obtain the latitude and longitude directly from a surveyor or from a licensed professional that obtained it from a surveyor?"
  - [ ] Yes: Coordinates are surveyor-obtained (GPS, total station)
  - [ ] No: Coordinates from mapping tool, Google Earth, or similar (less accurate)
- [ ] **Answer consistent with methods documented**:
  - [ ] If "Yes": Section C should reference GPS or survey control
  - [ ] If "No": Note source in Section D comments (e.g., "Coordinates from Google Earth")

### Section D Comments

Comments are a critical part of the EC. Verify the following are addressed:

- [ ] **Mechanical and Electrical systems described**:
  - [ ] Types of equipment measured listed
  - [ ] Locations noted (interior, exterior, attic, pad, etc.)
  - [ ] C2.e elevation justified by description
  - **Example**: "M&E include furnace in attic (252.3 ft), water heater in basement (234.1 ft), and exterior A/C pad (235.0 ft). Lowest is water heater at 234.1 ft."
- [ ] **Datum conversion explained (if applicable)**:
  - [ ] If Section C datum differs from BFE datum, conversion documented
  - [ ] Source cited (NOAA, USGS publication, NGS conversion)
  - [ ] Conversion direction clear (e.g., "BFE in NGVD 29; elevations converted using NOAA factor of +0.2 feet")
- [ ] **Any deviations from standard building diagram noted**:
  - [ ] If building does not match diagram perfectly, explain (e.g., "Diagram 7, but sunken living room is 2 feet below first floor level")
  - [ ] Measurement clarifications provided
- [ ] **Special conditions documented**:
  - [ ] Crawlspace flooding history or moisture staining
  - [ ] Blocked or missing flood vents
  - [ ] Unusual building features (split-level, breakaway wall, manufactured home)
  - [ ] Construction barriers (could not access basement, garage locked, etc.)
- [ ] **Photographs noted**:
  - [ ] Statement confirming photos are attached (required)
  - [ ] Number of photos (typically 2–4)
  - [ ] View directions noted if not obvious
- [ ] **Field condition assessment** (where relevant):
  - [ ] Crawlspace condition (dry, damp, flooded before)
  - [ ] Evidence of prior flood damage (staining, debris)
  - [ ] Structural concerns or items noted for owner awareness
- [ ] **Comments are legible** and professional in tone
- [ ] **Comments address most critical or unusual aspects** of the property

### Document Completeness

- [ ] **All pages present**:
  - [ ] Section A (Building and Property Information)
  - [ ] Section B (FIRM and Flood Data)
  - [ ] Section C (Elevation Information)
  - [ ] Section D (Certifier and Comments)
  - [ ] Photographs (minimum 2, ideally 4)
  - [ ] Any conversion factor documentation or reference maps
- [ ] **No pages are blank or missing** critical sections
- [ ] **Property address repeated** on all main pages (header or footer)

### Legibility and Professional Presentation

- [ ] **Form is legible** — Not faded, smudged, or hard to read
- [ ] **Handwriting (if any) is clear** and professional
- [ ] **Printed form is FEMA Form 086-0-33** (current version)
  - *Note: Form was updated in 2013; older versions should not be used*
- [ ] **Checkboxes clearly marked** (not ambiguous between two options)
- [ ] **No correction fluid or strikeouts** (redlines acceptable with initials and date)
- [ ] **Photographs are clear** and appropriately mounted or scanned
- [ ] **Overall presentation reflects professional firm standards**

### No Blank Required Fields

- [ ] **Every required field in Sections A–D has an entry**:
  - [ ] No "TBD", "Unknown", or blank fields unless marked "N/A" (not applicable)
  - [ ] If truly unknown, document reason and limitation in Section D
- [ ] **All C2.a–h elevation fields** have values or clear "N/A" notation
- [ ] **No assumptions** made without notation in comments

### Elevation Reasonableness Final Check

- [ ] **Floor-to-floor height is realistic** (8–10 feet for residential)
- [ ] **First floor is typically above BFE** (unless unusual circumstance documented)
- [ ] **M&E elevation makes sense** (below or at first-floor level)
- [ ] **Grade elevations bracket first-floor elevation** (LAG < C2.a < or ≈ HAG)
- [ ] **No transposition errors** (compare field notes to final values)
- [ ] **Datum is consistent throughout** (all elevations in same datum as BFE, or clearly converted)

### Flood Opening Validation (if applicable)

- [ ] **Number of openings** documented in field notes and form
- [ ] **Net open area calculation** verified:
  - [ ] Gross opening dimensions in field notes
  - [ ] Net open % estimated (account for grates, screens, louvers)
  - [ ] Math checked: Gross × Net % = Net area (in square inches)
- [ ] **Minimum requirements met** (if non-engineered):
  - [ ] ≥2 openings per side (if enclosure in floodplain)
  - [ ] Minimum net open area per ASCE 24 or local code (typically 1 sq in per 1 sq ft enclosure area)
- [ ] **If engineered**: Design documentation available or referenced

## Common Issues and Resolution Guide

### Issue: C2.a Measurement Inconsistent with Crawlspace Depth

**Problem**: First-floor elevation reported, but crawlspace appears much shallower than measured.

**Resolution**:
- Verify measurement was taken to **top of rim joist**, not top of floor joists
- Re-measure crawlspace if available
- Document in Section D if remeasurement confirms lower value
- If error found: Correct C2.a and note correction in Section D with date

### Issue: BFE Datum and Section C Datum Mismatch

**Problem**: Section B shows BFE in NGVD 29, but Section C elevations are in NAVD 88 (or vice versa).

**Resolution**:
- Confirm which datum is correct for the property
- If both are valid, apply published conversion factor
- Document conversion in Section D (source, direction, value)
- Ensure all C2.a–h values are consistently in stated datum

### Issue: LAG/HAG Are Very Close or HAG < LAG

**Problem**: Highest adjacent grade is lower than lowest adjacent grade (impossible).

**Resolution**:
- Check that LAG and HAG are not reversed
- Verify field notes for where HAG was measured
- For flat properties, LAG and HAG may be within 0.5 feet (acceptable)
- If property slopes, re-measure high point
- If measurement error confirmed, correct and document

### Issue: Flood Opening Net Open Area Exceeds Total Building Enclosure

**Problem**: Net open area calculated as greater than enclosure square footage (impossible).

**Resolution**:
- Verify net open area is in **square inches**, not square feet
- Recalculate: Gross opening (sq in) × Net % = Net open area (sq in)
- Confirm opening sizes are reasonable for foundation/crawlspace vents
- If error found, re-measure or correct calculation and note in Section D

### Issue: Photographs Do Not Show Foundation or Openings Clearly

**Problem**: Photos are too distant, too dark, or do not show required details.

**Resolution**:
- If available, request additional photos from field crew
- If time/access permits, re-visit site for better photos
- Document limitation in Section D (e.g., "Additional photos not available; foundation visible in side photos")
- If photos are missing entirely, EC cannot be released; request field re-visit

### Issue: Certifier License Number Does Not Match Name

**Problem**: Name is John Smith, but license shows Sally Johnson.

**Resolution**:
- Verify certifier is correct person (may be married name change, nickname, etc.)
- If discrepancy cannot be resolved, request correction from certifier
- Do not release EC until resolved (liability and regulatory issue)

### Issue: Section D Comments Are Blank or Minimal

**Problem**: Very short or no explanatory text in Section D.

**Resolution**:
- Require certifier to complete comments including:
  - Description of M&E systems measured and their elevations
  - Justification for diagram selection
  - Notation of any unusual findings
  - Source of datum conversion (if applicable)
- EC should not be released without adequate comments
- Comments document certifier's professional judgment and support accuracy of EC
